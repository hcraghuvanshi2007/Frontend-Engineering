# React is an library for building user interfaces. 
// It was developed by Facebook and is widely used for creating web applications. React allows developers to create reusable UI components, making it easier to manage and maintain complex applications.
# framework vs library
// A library is a collection of pre-written code that developers can use to perform common tasks. It provides specific functionality and can be used in various projects. Examples of libraries include jQuery, Lodash, and Axios.
// A framework, on the other hand, is a more comprehensive tool that provides a structure for building applications. It often includes libraries and tools to help developers create applications more efficiently [ 3rd party app]. Examples of frameworks include Angular, Vue.js, and Ruby on Rails.
# React is a library, not a framework. It focuses on the view layer of an application and allows developers to build user interfaces using components. React does not dictate how the rest of the application should be structured, giving developers more flexibility in how they organize their code and choose other libraries or tools to use alongside React.

# framework = necessary + unnecessary packages ( more memory)
# library = necessary packages

# Imperative vs Declarative programming
// Imperative programming (jQuery, vanilla JS) focuses on how to achieve a specific outcome. It involves writing step-by-step instructions for the computer to follow. For example, in jQuery, you might write code that directly manipulates the DOM to show or hide elements based on user interactions. [ ]

// Declarative programming (React) focuses on what the desired outcome is, rather than how to achieve it. In React, you describe the UI you want to see, and React takes care of updating the DOM to match that description. For example, you might write a React component that returns JSX to define the structure of the UI, and React will handle the rendering and updating of the DOM based on changes in state or props. # Data Binding 
    --  the flow of data between parent and child components.
// React uses one-way data binding, meaning that data flows in a single direction from parent to child components. This makes it easier to manage and debug applications, as changes in the data are predictable and easier to track.
// Angular and Vue use two-way data binding, which allows for more flexibility in how data is updated and shared between components. However, this can also lead to more complex applications and make it harder to track changes in the data.

// One-way data binding means that data flows in one direction, typically from parent components to child components through props.
Parent components pass data down to child components via props.
Child components cannot directly modify the props they receive; they are read-only.

// Two-way binding in React refers to a bidirectional data flow where changes in the component's state automatically update the UI (view), and user interactions in the UI simultaneously update the state ( ex: forms)


# Real Dom - 
   -- if change in the specific part of the page, then the whole page will be reloaded.
- created by browser
# Virtual Dom -
   -- if change in the specific part of the page, then only that part will be reloaded.
- created by React

# Why we use React? ** IMPORTANT **
-- Builds Dynamic UIs Easily: React allows developers to create dynamic and interactive user interfaces with ease. Its component-based architecture makes it simple to manage and update the UI as the application state changes.
-- Reusable Components: React promotes the creation of reusable components, which can be shared across different parts of an application or even across different projects. This leads to faster development and easier maintenance.
-- Fast and Efficient: React uses a virtual DOM to optimize rendering performance. When the state of an application changes, React updates only the necessary components, resulting in faster updates and improved performance.
-- Strong Community Support: React has a large and active community of developers, which means there are plenty of resources, libraries, and tools available to help developers build applications more efficiently.
-- Developer Friendly: React's declarative syntax and component-based architecture make it easier for developers to understand and work with. It also has a rich ecosystem of development tools, such as React Developer Tools, which can help developers debug and optimize their applications.
