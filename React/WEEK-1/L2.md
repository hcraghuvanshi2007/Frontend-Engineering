# Virtual DOM 
When something change in UI :
1. React creates a new virtual DOM tree.
2. React compares the new virtual DOM tree with the previous one to identify what has changed.
3. React updates only the specific part of the real DOM that has changed, rather than reloading the entire page. This process is called reconciliation [1] and it helps to improve performance by minimizing the number of updates to the real DOM.

# Important Qn = Difference between Virtual DOM and Real DOM

# [1] Reconciliation is the process by which React updates the real DOM to match the virtual DOM. 
It involves comparing the new virtual DOM tree with the previous one to identify what has changed, and then updating only the necessary parts of the real DOM to reflect those changes. This process helps to optimize performance by minimizing the number of updates to the real DOM, which can be slow and resource-intensive.

  
  # Diff b/w diffing and reconciliation: 
 - Diffing is the process of comparing two virtual DOM trees to identify what has changed, while reconciliation is the overall process of updating the real DOM to match the virtual DOM, which includes diffing as one of its steps.
    Whereas diffing focuses on identifying changes between virtual DOM trees, reconciliation encompasses the entire process of updating the real DOM based on those changes, including diffing, batching updates, and applying changes to the real DOM efficiently.
